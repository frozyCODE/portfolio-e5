Stage 1 : Automatique & Industrie

Le Contexte & L'Environnement Technique

  • Projet : Suite applicative HoloNet (applications Django-D et Yan Solo).
  • Cadre de travail : Équipe digitale, Méthode Agile Scrum (points quotidiens, fins de sprints).
⬇️

La Problématique (Le Constat)

  • Défauts UI/UX : Tableaux illisibles, navigation complexe (52 checkboxes), données de contrats inversées.
  • Risques techniques : Dépendance à des fichiers Excel manuels, calculs financiers sensibles (TVA) codés en dur (Magic Numbers).

Les Solutions Apportées

  • Interface : Refonte responsive avec Tailwind CSS (page unique), intégration du composant USelectMenu trié.
  • Modernisation : Connexion à l'API Bound pour récupérer automatiquement les BU (tests validés avec Postman).
  • Fiabilisation : Intégration de Decimal.js pour sécuriser les calculs (TVA, restants dus).

Le Bilan & Les Acquis

  • Bilan organisationnel : Outils financiers stables, interface ergonomique, gain de temps.
  • Bilan personnel : Pratique du workflow collaboratif (revues de code, Pull Requests).
  • Montée en compétences : Écosystème Vue.js/Nuxt et gestion de données rigoureuses.

Stage 2ème année

Laboratoire TIMC - CNRS / UGA

Stagiaire Développeur Java - Architecture Logicielle & Données

⬇️

Contexte

Développement de "CardiacTS", une application métier dédiée aux chercheurs en imagerie cardiaque.

Problématique

Concevoir une architecture logicielle capable de respecter le Plan de Gestion des Données (PGD) de la recherche publique tout en gérant des volumes massifs de données médicales.

Solution apportée

Création d'une application en Java et JavaFX, couplée à une infrastructure de données s'appuyant sur Docker et MinIO.
